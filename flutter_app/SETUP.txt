BrainTrain Flutter 뼈대 (flutter_app/)

=== Windows에서 "flutter 인식 안 됨"일 때 ===
저장소 루트(D:\AppProjects\BrainTrain)에서 PowerShell:

  powershell -ExecutionPolicy Bypass -File .\tools\install_flutter_windows.ps1

- Flutter stable SDK를 flutter_sdk\ 폴더에 받고, 사용자 PATH에 ...\flutter_sdk\flutter\bin 을 추가합니다.
- 설치 후 터미널/IDE를 한 번 닫았다가 다시 열어야 "flutter"가 잡힐 수 있습니다. Cursor를 쓰는 경우 Cursor 자체를 한 번 종료했다가 열면 PATH가 반영되는 경우가 많습니다.

PATH 없이 바로 실행 — 저장소 루트에서 (어느 폴더에서든 루트 경로만 맞추면 됨):

  flutter_bundle.cmd --version
  cd flutter_app
  ..\flutter_bundle.cmd pub get
  ..\flutter_bundle.cmd run -d windows

PATH 없이 — flutter_app 폴더에서:

  flutter_local.cmd run -d windows
  flutter_local.cmd doctor
  flutter_local.cmd pub get

  ※ PowerShell에서 .ps1 이 막히면(실행 정책): 위처럼 .cmd 를 쓰거나
     powershell -ExecutionPolicy Bypass -File .\flutter_local.ps1 run -d windows

또는 한 줄(PowerShell):

  $env:Path = "D:\AppProjects\BrainTrain\flutter_sdk\flutter\bin;$env:Path"; flutter run -d windows

=== Windows 데스크톱 실행이 "symlink" / Developer Mode 오류일 때 ===
Flutter(플러그인 포함)는 Windows에서 빌드 시 심볼릭 링크가 필요합니다.
설정에서 「개발자 모드」를 켜 주세요.

  Win + R → ms-settings:developers 입력 → Enter
  또는: 설정 → 개인 정보 보호 및 보안 → 개발자용 → 개발자 모드 켜기

PC 재시작은 보통 필요 없습니다. 적용 후 새 터미널에서:

  cd D:\AppProjects\BrainTrain\flutter_app
  .\flutter_local.ps1 run -d windows

※ Chrome/Edge(web)만으로는 이 프로젝트가 동작하지 않습니다. Drift(SQLite)가
  웹에서 dart:ffi를 쓰기 때문입니다. Windows/Android/iOS 등 네이티브 대상을 쓰세요.

※ Android 에뮬이 API 25 등으로 "unsupported"이면, AVD를 API 29+ 로 새로 만든 뒤
  flutter run 하면 됩니다.

=== Android 빌드: Kotlin "different roots" (C: Pub 캐시 vs D: 프로젝트) ===
증분 컴파일이 Pub 캐시(C:)와 프로젝트(D:)의 경로를 한 번에 묶지 못해
  this and base files have different roots ... Constants.kt ... flutter_app\android
  같은 스택이 나올 수 있습니다. android/gradle.properties 에 kotlin.incremental=false 가
  들어 있으면 완화됩니다. 근본적으로는 PUB_CACHE 를 D: 등 프로젝트와 같은 드라이브로 두거나,
  프로젝트를 C: 로 옮기는 방법도 있습니다. 한 번 꼬였으면: flutter clean 후 다시 빌드.

=== Firebase (firebase_core + Analytics) ===
- pubspec: firebase_core, firebase_analytics. bootstrap 에서 Firebase.initializeApp 후
  MaterialApp 에 FirebaseAnalyticsObserver 가 연결되어 화면 전환 로그가 수집됩니다(콘솔에서
  Analytics 활성화 필요).
- Android: android/app/google-services.json + app/build.gradle.kts 의 google-services 플러그인.
- lib/firebase_options.dart 의 Android 값은 google-services.json 과 일치합니다.
- 루트 .firebaserc / firebase.json : 기본 프로젝트 mybrainhealth 및 Android appId 연동 메타
  (firebase CLI·flutterfire 실행 시 참고). iOS/Web 앱은 콘솔 등록 후 plist·options 갱신 필요.
- Firebase 초기화가 실패해도 앱은 기동하며(로그만 출력), Analytics 옵저버는 앱이 등록된 뒤에만 붙습니다.

--- iOS / Web 의 GOOGLE_APP_ID 가 placeholder 인 경우 (콘솔에서 앱 추가 후 갱신) ---
1) Firebase CLI 로그인: npm i -g firebase-tools  후  firebase login
2) FlutterFire CLI (PATH 에 Pub 캐시 bin 추가 권장: %LOCALAPPDATA%\\Pub\\Cache\\bin):
     dart pub global activate flutterfire_cli
3) flutter_app 폴더에서:
     dart pub global run flutterfire_cli:flutterfire configure --project=mybrainhealth
   대화형으로 iOS·Web 등을 선택하면 lib/firebase_options.dart 가 덮어씌워집니다.
   (CLI 없이) 콘솔에서 iOS 앱 등록 후 GoogleService-Info.plist 만 받아 ios/Runner/ 에 넣고,
   콘솔에 표시된 iOS 앱 ID 로 firebase_options.dart 의 ios/macos 항목을 수동 맞출 수도 있습니다.

--- flutterfire 가 "Found 0 Firebase projects" 면 ---
  firebase login 이 안 된 상태입니다. 위 1)~3) 순서로 진행하세요.

=== AdMob (google_mobile_ads) ===
- 앱 ID: AndroidManifest + iOS Info.plist 의 GADApplicationIdentifier (동일 값).
- 광고 단위 ID: lib/core/ads/ad_unit_ids.dart (전면·배너)
- 메인 탭: 하단 내비 바로 위에 적응형 배너(ShellBottomBanner).
- bootstrap 에서 MobileAds.initialize 후 전면을 미리 로드합니다.
- 첫 리포트 탭 진입 시 전면 1회 노출 시도(같은 실행에서 한 번만). 디버그·에뮬에서는
  Google 권장 테스트 광고 단위로 바꾸는 것이 안전합니다.
- Windows 데스크톱 빌드는 Visual Studio 2022(v143) 및 CMake 3.22+ 가 없으면 링크/CMake 오류가
  날 수 있습니다. Android 전용으로 개발할 때는 에뮬만 쓰면 됩니다.

=== Windows: 로컬 AI 답변 읽기(TTS) — NuGet 불필요 ===
이전에는 flutter_tts 플러그인이 nuget.exe를 요구했습니다. 현재는 Windows에서
PowerShell(System.Speech)로 읽어 주며, 별도 NuGet 설치가 필요 없습니다.
(Android/iOS는 네이티브 TTS MethodChannel 사용)

- pub get만: powershell -ExecutionPolicy Bypass -File .\run_pub_get.ps1

1) Flutter SDK 설치 후 이 폴더에서 플랫폼 폴더가 없으면 한 번 생성합니다.

   flutter create . --project-name braintrain_flutter

2) 패키지 설치

   flutter pub get

3) Drift 코드 생성 (필수 — *.g.dart 생성)

   dart run build_runner build --delete-conflicting-outputs

   개발 중 자동 재생성:

   dart run build_runner watch --delete-conflicting-outputs

4) 실행

   flutter run

참고: Drift 생성 파일(app_database.g.dart, daily_condition_dao.g.dart, test_result_dao.g.dart)은 3번 실행 후 생성됩니다.

앱 구조 (요약)
- lib/main.dart → bootstrap() → ProviderScope + BrainTrainApp
- lib/app/brain_train_app.dart : 온보딩 여부 로딩 후 OnboardingPage 또는 MainShell
- lib/features/shell/main_shell.dart : 하단 탭(홈·측정·훈련·리포트·마이)
- lib/features/home, tests, training, report, mind_check, onboarding, mypage
- Drift 로컬 DB + MindInsightEngine / TodayAnalysisService
